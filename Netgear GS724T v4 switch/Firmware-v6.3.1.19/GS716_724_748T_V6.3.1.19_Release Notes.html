Fixed Issues:

  • Multicast packet with destination MAC address of 01:80:c2:00:00:1c and Ethernet type of 0x88CC causes loop in properly converged STP topology.
  • Debugging URL of ‘devshell’ is accessible without authentication.
  • Can't correctly set VLAN Tag in 'Port PVID Configuration' GUI page.
  • Switch incorrectly discards IS-IS hello packages.

Modifications:

  • Added configurable option “Block Unknown Multicast Address” in GUI page of ‘Switching > Multicast > IGMP Snooping > IGMP Snooping Configuration’ to Enable or Disable unknown multicast traffic flooding.
    • The default is disable, and switch floods unknown multicast traffic.
    • When it is enabled switch blocks un-registered multicast traffic.

Known issues:

  1. Warning messages pop up when the HTTPS admin status is enabled.
    [Workaround]: Refresh the GUI screen.
  2. The cable test results sometimes show a much longer value.
    [Workaround]: Redo the cable test on the same port.

Notes :

  • The released zip contains switch firmware (gs7xxt-v6.3.1.19.stk) and MIB files. Please unzip the file to extract the switch image for firmware upgrade.
  • During the boot time (about 2 minutes), all LED lights on for GS716Tv3 and GS724Tv4. This is working as HW design.

Limitations:

  1. It is NOT supported for configuration migration from older devices GS716Tv2/GS724Tv3/GS748Tv4 to new devices GS716Tv3/GS724Tv4/GS748Tv5.
  2. The boot time for GS716Tv3/GS724Tv4/GS748Tv5 is about 40 seconds longer than GS716Tv2/GS724Tv3/GS748Tv4.

Upgrade Procedure:

  1. Download the new firmware 6.3.1.19 from NETGEAR product support Website and store it in local PC, and remember its location in the PC's file system. Then unzip it to get the switch image "gs7xxt-v6.3.1.19.stk".

      Download Link: http://www.downloads.netgear.com/files/GDC/GS716TV3/GS716_724_748T_V6.3.1.19.zip
  2. There are four methods to upgrade the new firmware from PC to the switch.
    1)     HTTP
    2)     TFTP
    3)     USB
    4)     SCC (Smart Control Center)
    Below are the steps to upgrade the firmware via HTTP, which is the most convenient method. For the other three methods, please check the IG and SW User Manual for details.
  3. To use HTTP method, go to GUI page "Maintenance > File Management > Dual Image > Dual Image Configuration", find the backup (next-active) image.
  4. Suppose the backup-image is image1, then go to “Maintenance > Download > HTTP File Download”, set File Type as ‘Archive’, set Image Name as ‘Image1’, select the firmware file ‘gs7xxt-v6.3.1.19.stk’ stored in local PC via Browser button, then click Apply button to start firmware downloading.



    The download will take several minutes. Do not power cycle the switch before and the success message shows up as below.

  5. Go to GUI page "Maintenance > File Management > Dual Image > Dual Image Configuration", set 'Image Name' with Image1, check the checkbox of 'Activate Image', then Apply.


     
  6. Go to GUI page "Maintenance > File Management > Dual Image > Dual Image Status", to double check 6.3.1.19 is correctly downloaded to image1 and image1 is the 'next-active' image. If not, go back to step 3 to download the switch image again.

  7. Go to GUI page 'Maintenance > Reset > Device Reboot', set 'Reboot Unit No.' as 'All' (default option); check the checkbox then click Apply button.



    The restart may take several minutes, do not power cycle or reboot the switch before the process is complete.
  8. After the switch is restarted, login the switch to double check whether the FW is upgraded correctly, via GUI page of "GUI page System > Management > System Information".
